All-In-One Condiment Dispenser

The newly innovated kitchen tool that will help users store 4 condiments in one package, dispense the condiment in a controlled amount and improve confidence in following recipes for those home cook masters!

4 Condiments Dispensable

The device is able to store and dispense up to 4 different condiments without contamination

Amount Dispensed Measurable

The electronics can measure how much each condiment being dispensed and display its weight via the screen. Great for following recipes!

Food Safe & Cleanable

The product is designed to be safe and detachable for filling the condiments and cleaning. It is also made from food safe materials

Overall Body
Made from food-safe plastics with ergonomic design
Double-Motor System
The first motor rotate the mechanism at top of the device to select the correct condiment demanded to be dispensed. The second motor pushes the selected condiment container to dispense through the valve
Battery powered
Make the device portable and more convenient since no wires are attached
Valves with spring system
The valves prevent the condiments from being dispensed when not in use. they also allow the condiments to be dispensed when the condiment container is pushed down. The spring is there to reset the position of the container when the operation is completed
Digital Screen
Display the amount and type of condiment being expensed through the screen
Detachable condiment container
Detachable for refilling and cleaning

About Us

The team was formed from a group of Imperial students that enjoy cooking but own nothing more than a tiny kitchen. A problem arised when more condiments were bought but there was no space to store. Moreover, without a weighting scale, it was a quite a challenge to follow the exact recipe when learning how to cook new dishes

From the initial research, we found that there was very little development in products that can solve the problems specifically for those who own tiny kitchen, this made us want to be the pioneer innovators to fix the problems

From discussion, we concluded that it was the right time to revolutionise the cooking experience, and make a newly innovative product to make the task more convenient, more economical use of space and more efficient use time

Inspired by bar taps, we suggest designing a multi-functional condiment dispenser that helps users select, measure and dispense different spices all at once. This not only saves the hassle created by confusing different spices but also saves the user kitchen space

Steps for registering and setting up the domain name for the website

To set up a www or custom subdomain, such as www.example.com or blog.example.com, you must create a CNAME file in your site's repository and configure a CNAME record with your DNS provider. Below are the follwing steps

1. On GitHub, navigate to your site's repository

2. Under your repository name, click Settings

3. Under "Custom domain", type your custom domain, then click Save. This will create a commit that adds a CNAME file in the root of your publishing source.

4. Navigate to your DNS provider and create a CNAME record that points your subdomain to the default domain for your site. For example, if you want to use the subdomain www.example.com for your user site, create a CNAME record that points www.example.com to .github.io. For more information about how to create the correct record, see your DNS provider's documentation. For more information about the default domain for your site, see "About GitHub Pages."

5. Open Git Bash.

6. To confirm that your DNS record configured correctly, use the dig command, replacing WWW.EXAMPLE.COM with your subdomain.

7. If you use a static site generator to build your site locally and push the generated files to GitHub, pull the commit that added the CNAME file to your local repository. For more information, see "Troubleshooting custom domains and GitHub Pages."

8. Optionally, to enable HTTPS encryption for your site, select Enforce HTTPS. It can take up to 24 hours before this option is available. For more information, see "Securing your GitHub Pages site with HTTPS."
